# CoRe IR for `arm64vm`

`arm64vm` can run **either**:

1) ARM64 assembly (`.s`) generated by `forge` (default VM mode) / `forge -v` (or legacy `forge --asm`), **or**
2) A simplified, text-based **CoRe VM IR** (this file).

This IR is meant to be *easy to write by hand* and *easy for the VM to parse*, while still supporting:
- functions + calls
- locals/temps
- loops + branches (labels, jumps)
- numbers/strings/bools/null
- lists + maps
- a few builtins (`len`, `keys`, `values`, `range`)

## Running IR

```bash
arm64vm --ir program.cir
# or just:
arm64vm program.cir
```

Print a starter template:

```bash
arm64vm --sample-ir
```

## Two accepted formats

### 1) Section mode (recommended)

```text
fn add.params: a, b
fn add.actions:
  add out a b
  ret out

fn main.params:
fn main.actions:
  const x 2
  const y 3
  call z add x y
  print z
  ret 0
```

Rules:
- `fn <name>.params:` takes a comma-separated param list.
- `fn <name>.actions:` starts an **indented** block (2 spaces or a tab).
- A non-indented `fn ...` line starts a new section.

### 2) Brace mode

```text
fn main: {
  const i 0
loop:
  lt c i 5
  jz c done
  add i i 1
  jmp loop
done:
  ret 0
}
```

## Comments

- `# comment`
- `// comment`

Comments are ignored (string literals keep `#` and `//` inside quotes).

## Literals

- Numbers: `10`, `3.14`, `-5`
- Booleans: `true`, `false`
- Null: `null`
- Strings: `"hello world"` (supports escapes `\"`, `\\n`, `\\t`, `\\r`, `\\\\`)

## Locals

Any token that isn’t a literal is treated as a local variable name (locals default to `null` if read before set).

## Instruction set

### Data / arithmetic

- `const <dst> <literal>`
- `mov <dst> <src>`
- `add <dst> <a> <b>` (number+number, or string concat)
- `sub <dst> <a> <b>`
- `mul <dst> <a> <b>`
- `div <dst> <a> <b>`

### Compare / boolean

- `eq <dst> <a> <b>`
- `ne <dst> <a> <b>`
- `lt <dst> <a> <b>`
- `gt <dst> <a> <b>`
- `and <dst> <a> <b>` (truthy)
- `or  <dst> <a> <b>` (truthy)
- `not <dst> <a>`

Truthy rules: `false/null/0/""/[]/{}` are falsey; everything else is truthy.

### Control flow

- `label <name>` is supported as sugar via `<name>:` on its own line
- `jmp <label>`
- `jnz <cond> <label>` (jump if truthy)
- `jz  <cond> <label>` (jump if falsey)

### Functions

- `call <dst|_> <func> <arg1> <arg2> ...`
- `ret [value]`

Use `_` to ignore a return value:

```text
call _ doSomething x
```

### Output

- `print <value>` (prints one line)

### Lists

- `list <dst> [items...]`
- `idxget <dst> <list> <index>` (supports negative index)
- `idxset <list> <index> <value>` (supports negative index)

### Maps

- `map <dst>`
- `mset <map> <key> <value>`
- `mget <dst> <map> <key>`

Keys are coerced to strings for `null/number/bool/string`.

## Builtins

Called via `call`:

- `call out len x`
- `call out keys m`
- `call out values m`
- `call out range start end` (end-exclusive)

## What this IR is *not*

- It is **not** the full CoRe language parser.
- Expressions are intentionally minimal (one token per operand).
- It’s designed to be a stable “VM input format” that we can evolve without breaking VM mode (`forge` / `forge -v`, and legacy `forge --asm`).
