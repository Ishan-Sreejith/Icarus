## 🐛 ISSUE DISCOVERED

**Problem**: Function calls in the JIT compiler crash or return incorrect values

**Test Cases**:
1. ✅ Basic arithmetic (no functions): Works perfectly (5+3=8, 8*2=16)
2. ❌ Function with parameters: Returns garbage or wrong values
3. ❌ Function without parameters: Crashes with panic

---

## 🔍 ROOT CAUSES IDENTIFIED

### Issue 1: Return Value Corruption
**Location**: src/jit/compiler.rs, Return instruction handler (lines 484-502)
**Problem**: The retain/release runtime calls overwrite the return value in x0

**Original Code**:
```rust
IrInstr::Return { value } => {
    if let Some(val) = value {
        let loc = self.regmap.get(val).unwrap();
        emit.move_to_phys_reg(0, loc);
        emit.emit_call(RuntimeFunctions::retain().addr());  // ← OVERWRITES x0!
    }
    // ... cleanup that also overwrites x0 ...
    emit.emit_u32_le(encode_ldp_fp_lr());
    emit.emit_u32_le(encode_ret());
}
```

**Fix Applied**: Commented out retain/release calls temporarily
- Return values now preserved in x0
- TODO: Re-implement with proper value preservation

### Issue 2: Missing Variable in Register Map
**Location**: src/jit/compiler.rs, Move instruction (line 278)
**Problem**: Variables from function calls not found in register map

**Symptoms**:
- Panic: `called Option::unwrap() on None value`
- Happens when trying to use function return value

**Likely Cause**: The Call instruction handler allocates a register but may not properly record it, or the IR generates intermediate variables that aren't tracked

---

## 🔧 FIXES APPLIED

### 1. Disabled Retain/Release in Return (TEMPORARY)
**File**: src/jit/compiler.rs, lines 484-503
**Status**: ✅ Applied
**Result**: Functions no longer corrupt x0, but still have issues

### 2. Better Error Messages
**File**: src/jit/compiler.rs, lines 275-281, 466-475
**Status**: ✅ Applied
**Changes**:
- Move instruction: Better error when variable not found
- Call instruction: Better error for missing function arguments
